# Выгрузка ордеров PayOut

### Описание

Метод используется для получения ордеров PayOut с их подробными данными за определенный диапазон времени. Все параметры передаются в query (не в теле)

-------------------
### Структура запроса

* Метод: `GET`
* Endpoint: `https://domain/public/api/v1/shop/payout-orders`
* Headers: `"Authorization": "{Bearer API_KEY}"`

-------------------

### Параметры запроса (передаются в query)

| Поле              | Обязательное поле| Object| Тип            | Описание                                       | Пример заполнения |
| ----------------- | ---| -------------------| ---------------| ------------------------------------------------ | --------------------- |
| `from`          | Обязательно| Блок               | `float`        | Начало периода (YYYY-MM-DD)                                   |    2025-08-01                   |
| `to`              | Обязательно| customer           | `string`       | Конец периода (YYYY-MM-DD)                                   |  2025-09-01                     |
| `status`            | Тип| payment            | `string`       | Статус ордера | `any-bank`            |
| `page`        | Опционально| Блок               | `string`       | Номер страницы (по умолчанию: 1)    | `RUB`                 |
| `take`            | Опционально| payment            | `string`       | количество записей на страницу (опциональный, по умолчанию: 100, максимум: 1000)            |


-------------------

### Поддерживаемые значения в полях

<details>

<summary><strong>Поддерживаемые значения <code>status</code></strong></summary>

| Значение           | Описание                                                                 |
| ------------------ | ------------------------------------------------------------------------ |
| `new`              | Новый                    |
| `requisites`        | В поиске реквизитов                                                |
| `customerConfirm`       | Реквизиты найдены. ожидается платеж от пользователя                                 |
| `traderConfirm`             | Пользователь выполнил платеж, ожидается подтверждение трейдера |
| `completed`          | Трейдер подтвердил входящий платеж                                                          |
| `cancelled`     | Отменен                                               |
| `error`   | Ошибка произошедшая на любом из этапов            |
| `dispute`             | Диспут                                        |

</details>

Реализовано
Разрешено передавать параметры from, to в форматах
1. 2025-12-19T00:00.000+02:00 - cо смещением от UTC (Нужно обратить внимание, что символ + будет проигнорирован, поэтому необходимо вместо символа + ввести %2b )
2. 2025-12-19T00:00.000Z - в UTC
3. 2025-12-19 - старый формат для обратной совместимости
По умолчанию время будет в часовом поясе UTC
Время по умолчанию будет 00:00:00.000
Если передать дополнительный параметр в фильтре tz, то время в формате 2025-12-19 будет установлено для соответствующего часового пояса, например tz=europe/moscow
