---
title: "Статусы ордера"
---

В каждый момент времени ордер может иметь только один статус (`status`). Некоторые статусы могут иметь детали (`statusDetails`).

## Основные статусы

| Статус            | Описание                                                                            |
| ----------------- | ----------------------------------------------------------------------------------- |
| new               | Ордер создан                                                                        |
| requisites        | Клиент выбрал метод платежа, система ищет подходящие реквизиты трейдеров для ордера |
| customer\_confirm | Система нашла трейдера для ордера и ожидает оплаты клиентом                         |
| trader\_confirm   | Клиент подтвердил оплату, ожидание подтверждения трейдером                          |
| completed         | Трейдер получил перевод                                                             |
| cancelled         | Ордер отменен                                                                       |
| dispute           | Ордер переведен в диспут, может быть завершен или отменен только через поддержку    |
| error             | Произошла ошибка в процессе обработки ордера, требуется обращение в поддержку       |

## Детали статусов

### Статус `dispute`

| Detail                   | Описание                                                                         |
| ------------------------ | -------------------------------------------------------------------------------- |
| no\_payment              | Оплата не пришла                                                                 |
| different\_amount        | Пришла другая сумма                                                              |
| admin\_created           | Ордер создан админом                                                             |
| revert\_cancelled        | Ордер возвращен из отмененного состояния                                         |
| trader\_confirm\_timeout | Вышло время подтверждения оплаты трейдером, есть загруженная квитанция об оплате |

### Статус `cancelled`

| Detail                     | Описание                                                           |
| -------------------------- | ------------------------------------------------------------------ |
| shop                       | Ордер отменен магазином                                            |
| admin                      | Ордер отменен админом                                              |
| operator                   | Ордер отменен оператором                                           |
| customer                   | Ордер отменен клиентом                                             |
| trader                     | Ордер отменен трейдером                                            |
| new\_timeout               | Вышло время выбора платежного метода                               |
| requisites\_timeout        | Вышло время поиска реквизитов                                      |
| customer\_confirm\_timeout | Вышло время оплаты клиентом, квитанции нет                         |
| trader\_confirm\_timeout   | Вышло время подтверждения оплаты трейдером, квитанции нет          |
| sync\_requisites\_attempts | Ордер отменен из-за превышения числа попыток назначения реквизитов |

### Причины статусов (Order status reason)

| Reason                | Описание                         |
| --------------------- | -------------------------------- |
| different\_amount     | Сумма заказа была изменена       |
| fake\_receipt         | Загружена поддельная квитанция   |
| repeat\_payment       | Оплата была повторена            |
| verification\_expired | Верификация оплаты истекла       |
| waiting\_merchant     | Ожидание квитанции от мерчанта   |
| waiting\_trader       | Ожидание квитанции от трейдера   |
| wrong\_requisites     | Предоставлены неверные реквизиты |
