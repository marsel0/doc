# PayIn. Получение информации по заявкам за период


## Описание

Метод используется для получения списка заявок за период времени.


## Структура запроса

* Метод: `GET`
* Endpoint: `https://domain/public/api/v1/shop/orders`


## Header Parameters (Заголовки запроса)

| Параметр      | Обязательное | Тип    | Значение              | Описание                      |
| ------------- | ------------ | ------ | --------------------- | ----------------------------- |
| Authorization | Обязательно  | string | Bearer **shopApiKey** | Токен авторизации магазина |


## Query Parameters (передаются в URL)

| Параметр | Обязательное | Тип    | Описание                                                           | Пример заполнения |
| -------- | ------------ | ------ | ------------------------------------------------------------------ | ----------------- |
| from     | Да           | string | Начало периода (YYYY-MM-DD)                                        | 2026-02-01        |
| to       | Да           | string | Конец периода (YYYY-MM-DD)                                         | 2026-02-27        |
| page     | Нет          | string | Номер страницы (по умолчанию: 1)                                   | 2                 |
| status   | Нет          | string | Статус ордера                                                      | completed         |
| take     | Нет          | string | Количество записей на страницу (по умолчанию: 100, максимум: 1000) | 50                |


### Поддерживаемые значения в полях

<details>

<summary><strong>Поддерживаемые значения <code>status</code></strong></summary>

| Значение           | Описание                                                                 |
| ------------------ | ------------------------------------------------------------------------ |
| `new`              | Новый                    |
| `requisites`        | В поиске реквизитов                                                |
| `customerConfirm`       | Реквизиты найдены. ожидается платеж от пользователя                                 |
| `traderConfirm`             | Пользователь выполнил платеж, ожидается подтверждение трейдера |
| `completed`          | Трейдер подтвердил входящий платеж                                                          |
| `cancelled`     | Отменен                                               |
| `error`   | Ошибка произошедшая на любом из этапов            |
| `dispute`             | Диспут                                        |

</details>
