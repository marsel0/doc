### Описание
Метод используется для открытия диспута по заявке используя ID заявки. В query запроса добавляется параметр ID заявки.


## Структура запроса

* Метод: `POST`
* Endpoint: `$BASE_URL/public/api/v1/shop/orders/{id}/dispute`


## Header Parameters (Заголовки запроса)

| Параметр      | Обязательное поле | Тип    | Значение              | Описание                    |
|---------------|-------------------|--------|-----------------------|-----------------------------|
| Authorization | Да                | string | Bearer **shopApiKey** | Токен авторизации магазина  |
| Content-Type  | Да                | string | multipart/form-data      | Формат данных в теле запроса  |


## Path Parameters (передаются в URL)

| Параметр | Обязательное | Тип    | Описание  |
| ---- | ------------ | ------ | --------- |
| id   | Обязательно  | string | ID заявки |


## Request Body parameters (Параметры тела запроса)


| Параметр              | Обязательное поле | Object     | Тип    | Описание                                                         |
|-----------------------|-------------------|------------|--------|------------------------------------------------------------------|
| **amount**            | Да                | -          | number | Сумма заявки                                                     |
| file              | Да                | -          | string <binary>| Файл чека                                                    |


### Пример запроса

```bash
curl --location --request POST '$BASE_URL/public/api/v1/shop/orders/ac0b30c1-c1ca-4a99-a654-4c563788802d/dispute' \
--header 'Authorization: ••••••' \
--form 'file=@"/F:/receipt_example3.pdf"' \
--form 'amount="1040"'
```
